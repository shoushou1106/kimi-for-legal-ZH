# legal-profile/ —— 你的法律实务画像目录

本目录存放**你的个人实务画像**，由各领域「冷启动访谈」写入，全部领域工作流在执行前从这里读取。

- `company-profile.md` —— 共享公司画像（主体身份、业务、经营地域、风险偏好、关键人员、本地知识库路径），由 11 个领域共享。模板见 `../references/company-profile-template.md`。
- `<领域名>.md` —— 各领域实务画像（如 `commercial-legal.md`），模板见 `<领域名>/profile-template.md`。
- `<领域名>/` 子目录 —— 事项工作区（`matters/`）、验证日志、续约登记册等领域数据文件。

## 使用方式

1. 不要手工从零编写。先运行任一领域的冷启动访谈（对 KIMI 说"运行商事合同冷启动访谈"），它会通过访谈和种子文件生成画像。
2. 画像生成后可以直接编辑修正——所有工作流下次执行时自动读取新版本。
3. **本目录是你的个人数据，不要提交到 Git 仓库**（已在 `.gitignore` 中排除，本 README 除外）。KIMI Work 用户可选择性将画像要点摘要写入 KIMI 长期记忆，跨工作区生效。
